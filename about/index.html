About

Data

I used the tweepy Python library to gather twitter data. Tweepy provides access to the well documented Twitter API. Tweets are strutured using JSON, which is easily read using pandas and stored in dataframes. I used a corpus of the 10 highest [2016] paid male athletes, and female athletes, that is a total of 20 athletes, to filter streamed twitter data. For my initial results, submitted on May 1, where after running my script for around 8 hours provided ~560 MB of data which I used in this preliminary study. At this point I have collected around 2.02 GB data, collected over 3 days. The data-set has 167,322 unique entries, collected from 90 countries.

Sentiment Mining

I performed sentiment analysis and associated each entry with a positive, negative or neutral sentiment. The tweet text is analyzed a given a polarity from -1 to 1, with anything above 0 positive, below 0 negative, and everything else is neutral. A Naive Bayes analysis is used for classification on an n-gram model of the text. I used the TextBlob, which is a Python library for processing textual data utilizing the nltk library in Python. The classification is reported to have an accuracy of 0.8. Some examples of the classification:

  1. Positive: 11 years ago today, Kobe Bryant absolutely destroyed Steve Nash.
  2. Negative: They might've been worse than Iverson 2001 Sixers team...
  3. Neutral: Remember when Serena williams beat sharapova and a Nigerian paper's headline the next day was "Sharap it's ova"

Data Analysis

I used pandas to convert the data into dataframes allowing easier query, sort and statistical operations. I used matplotlib for plotting, scipy and numpy for data analysis.
